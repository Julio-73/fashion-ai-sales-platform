# Dashboard UI Standards

## SaaS Dashboard Shell

- Use a stable sidebar with clear product areas.
- Keep top bars reserved for tenant/company switching, search, notifications, and user actions.
- Keep page headers compact: title, short contextual metadata, and primary actions.
- Make the first viewport useful, not decorative.
- Avoid marketing hero patterns inside operational dashboards.

## CRM Interfaces

CRM screens should make pipeline work fast:

- pipeline/stage filters
- owner assignment
- lead/customer status
- next action
- last contact time
- deal value or sales potential
- bulk actions where useful
- quick detail side panels

## Analytics Dashboards

Analytics screens should include:

- KPI summary cards
- trend charts
- breakdown tables
- filter bars
- date ranges
- export/share actions when relevant
- clear empty states for no data

Prioritize chart clarity over decorative visuals.

## Conversation Interfaces

AI and WhatsApp conversation views should support:

- conversation list with filters and unread state
- active thread with message grouping
- customer context panel
- AI draft/assist area
- message composer
- assignment/status controls
- delivery/error indicators
- responsive split view behavior

## Product Catalogs

Catalog screens should support:

- searchable product grid/table
- SKU, category, collection, price, stock, size, and color filters
- quick edit actions
- image-safe card layouts
- bulk import/export states

## Automation Panels

Automation interfaces should expose:

- trigger
- conditions
- actions
- status
- recent runs
- errors
- enable/disable control
- safe preview or dry-run when possible
